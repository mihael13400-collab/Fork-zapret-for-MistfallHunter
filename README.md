### Запуск


1. Запустите скрипт `\utils\calc_ttl.bat`. 
Он автоматически выполняет трассировку маршрута до ip адреса из диапазона AWS (AS16509), который был выбран на основе анализа трафика игры Mistfall Hunter.
Важно: это ОЧЕНЬ приблизительный метод, так как серверные ip адреса могут динамически меняться, но он даёт примерное стартовое значение TTL.

2. Запустите стратегию вручную или через `service.bat`. Сценарий автоматически подтянет значение из файла `ttl.txt` и применит его для параметра `--dpi-desync-ttl`.

3. Если после применения стратегии наблюдаются разрывы соединения  или "вылетает" сервер, попробуйте вручную уменьшить значение TTL в файле `\utils\ttl.txt` и перезапустите стратегию.

## Контекст
Игровые сервера **Mistfall Hunter** работают на **AWS (AS16509)**.
В России трафик до AWS в некоторых регионах, может попадать под 16 кб блок.

Подробнее о механизме блокировки:
- [net4people/bbs#490](https://github.com/net4people/bbs/issues/490)
- [ntc.party](https://ntc.rkn.quest/t/%D0%B1%D0%BB%D0%BE%D0%BA%D0%B8%D1%80%D0%BE%D0%B2%D0%BA%D0%B0-cloudflare-ovh-hetzner-digitalocean-09062025-xxxxxxxx/17013)





Игра использует нестандартный бинарный протокол поверх TCP.
Первый пакет после TCP handshake - 79 байт, сигнатура `00 00 04 4D`.
На основе этого написан простой WinDivert фильтр.
